# 📋 Specification

These are the original Kiro spec documents that drove the implementation:

- **`requirements.md`** — User stories with EARS-format acceptance criteria.
  9 requirements covering ingestion, link scanning, message analysis,
  alerting, logging, weekly reports, autonomy, reasoning, and deployment.
- **`design.md`** — Architecture, component design, data models, technology
  choices, and 10 testable correctness properties.
- **`tasks.md`** — 12-task / 61-subtask implementation plan with property
  test markers.

If you want to extend this project, start with `design.md` to understand
the intended boundaries between the agents.
